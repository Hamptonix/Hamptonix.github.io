Security Model

  • AES-256-GCM encryption via the browser's native Web Crypto API — no JS libraries, no external deps
  • Your key is passed through PBKDF2 (310,000 iterations, SHA-256) to derive the actual crypto key — brute force is extremely costly
  • A random 128-bit salt and 96-bit IV are generated fresh for every encryption operation
  • The encrypted output contains: salt + IV + ciphertext — no key, no hints, no metadata
  • Inspecting the locked HTML in DevTools reveals only the ciphertext blob — the key and content are completely absent from the file
  • Decryption happens in memory only; the decrypted content is injected at runtime and never written to disk or storage
  • GCM authentication tag means a wrong key produces an explicit failure — no silent partial decryption
